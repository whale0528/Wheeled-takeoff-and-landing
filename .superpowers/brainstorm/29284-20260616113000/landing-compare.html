着陆轨迹方案对比

几何轨迹法 vs 闭环制导法 — 核心逻辑完全不同

方案 A：几何轨迹法（当前代码）
思路：在地面预先算好整条 4 段轨迹

步骤：
① 陡下滑线：H = -(X - X₀)·tan(γ₁)
② 圆弧过渡：圆心 + 半径几何推导
③ 浅下滑线：H = -(X - X_aim)·tan(γ₂)
④ 指数拉飘：H = H₀·e^(-t/σ)

输出：
H_cmd = f(X_current)
一张预计算的 X→H 映射表

❌ 接地位置固定，无法适应风/速度变化
❌ 需要手算圆弧切点和半径
方案 B：闭环制导法（A320 方案）
思路：传感器实时测量偏差，控制器闭环跟踪

步骤：
① GS下滑道：以跑道入口为基准，定义一条固定下滑角直线
② 实时计算高度误差 e_h
③ 俯仰跟踪路径，油门保持速度
④ 进入FLARE时记录 h₀, ḣ₀，计算指数下沉率轨迹

输出：
θ_c = f(h_err, ḣ_err)
实时闭环俯仰指令

✅ 自动适应风和速度变化
✅ 不需要预计算轨迹几何

关键公式对比

下滑道跟踪（替代几何轨迹的①②③段）

下滑路径高度：hpath = hthr + D · tan(3°)   (D = 距入口距离)
高度误差：eh = hpath - h
目标下沉率：ḣc = VG·tan(3°) + Kh·eh
俯仰跟踪：θc = γc + αtrim + Kḣ(ḣc - ḣ)

指数FLARE（替代几何轨迹的④段）

进入瞬间记录：h₀, ḣ₀, V₀, θ₀
时间常数：τ = h₀ / (|ḣ₀| - |ḣTD|)
目标下沉率：ḣc = -(h + hb) / τ
俯仰指令：θc = θ₀ + Kḣ(ḣc - ḣ) + Kaz(az,c - az)
